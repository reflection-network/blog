## Title

Agent Speaks

## Description

The agent gets a Telegram transport. Same config, different adapter import. One line changes, silent container becomes a chatbot.

## Key themes

- Three-layer architecture: capsule (who) → adapter (how) → runtime (what runs)
- Adapter pattern: same mkAgent interface, different backends
- Brain in a jar gets a voice: from isolated introspection to bidirectional communication
- Long-polling transport: Telegram Bot API, simple and stateless
- Offset=-1 flush: preventing message replay on restart (distributed systems lesson)
- One-shot invocation: single responses, no conversation history yet
