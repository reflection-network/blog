## Title

What if an AI Agent Was Just a Git Repo?

## Description

Every AI agent framework wants you to write code their way. What if the agent's identity just lived in a Git repo instead?

## Key themes

- Git as agent evolution: commits = snapshots of identity, branches = experiments, forks = reproduction
- Identity vs infrastructure: who the agent is vs how it runs
- Declarative config: code says HOW, config says WHAT
- Escape from framework lock-in: Git solves half the problems frameworks reinvent
- Capsule concept: a declaration of identity without implementation coupling
