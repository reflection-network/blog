## Title

Readable Repos

## Description

Seven repos, seven READMEs. Consistent format, no boilerplate. The project goes from 'read the Nix' to 'read the README' in one iteration.

## Key themes

- Documentation debt: working software without explanation looks abandoned
- READMEs as handshake: the first thing visitors see
- Uniform format across repos: H1 → one-liner → usage → config → architecture
- Two layers of docs: READMEs for quick reference, docs site for full walkthrough
- Consistency as documentation: familiar structure across repos
- Seven repos, zero → seven READMEs
