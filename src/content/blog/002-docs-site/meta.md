## Title

Docs From Day One

## Description

Most projects add docs when they're ready. That's backwards. A standalone Starlight site deployed at iteration two.

## Key themes

- Documentation as structural statement: a docs site says "this project takes docs seriously"
- Three public surfaces: landing page (pitch), docs site (how-to), GitHub (source)
- Zod v4 bug: a concrete debugging story with community workarounds
- Docs as design validation: if you can't explain it, the design isn't clean
- Separate concerns: different audiences for each surface

Three rectangular wireframe panels arranged side by side — different sizes, different internal structures, but sharing a common baseline. They represent three public surfaces (landing page, docs, source). Thin connection lines link corresponding points between the panels. The leftmost panel is the smallest and most dense, the middle one is open and structured like a table of contents, the rightmost is raw and grid-like.

No text, no labels, no UI elements — purely abstract.

Style: minimal, precise, engineering blueprint aesthetic, high contrast, lots of negative space, strict geometry.
